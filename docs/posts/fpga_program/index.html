FPGA多路编程器

  • May 13, 2023
  • Categories: 编程
  • #FPGA
  • One minute read

前言

FPGA是“现场可编程门阵列”，强调的就是“现场可编程”。当我们设计好FPGA，进入量产阶段，需要对成千上万的FGPA进行编程。本项目设计的多路FPGA编程器，可以同时对多达8路FPGA进行编程，成倍提高工作效率。

单次单路FPGA编程大约需要25秒左右，当我们扩展出8路，对8路进行并行编程，总共才需要不到28秒的时间，效率极高。

功能框图

  • TF卡，在PC上把FPGA的编程bin文件拷贝到TF卡上，让MCU读取；
  • SPI-FLASH和EEPROM，用于存储FPGA的编程bin文件；
  • LCD和按键，编程器的输入输出设备；
  • FPGA，实现多路编程的关键模块。

多路实现

FPGA的优势

对FPGA编程，使用的是SPI接口。要实现多路编程，就需要很多路SPI。显然，STM32没有这么多SPI可用，需要扩展。这正好是FPGA的强项。

我们可以把FPGA理解为可定制芯片，按需求对芯片进行定制。现场编程，现场可用。现在我们需要扩展SPI，就可以选一颗FPGA，通过verilog等语言设计出SPI接口，在FPGA内部复制出8路，实现扩展。这样，我们就可以在FPGA的辅助下，控制任意多的SPI口，而不受MCU本身接口资源的限制。这就是FPGA的优势所在。

分时复用效率不高

多路实现，一般采用的方法分时复用的方法。MCU分多次，分时分别对多路进行操作。这样可以节省一些操作，但节省的时间有限，效率提高不多。

FPGA并行优势

在本项目中，对FPGA的编程，大部分是下行数据，把bin文件写入到被编程的FPGA内部。其余是大量的等待时间，等待FPGA内部操作完成，继续下一步动作。少量上行数据，是读取状态，校验等。下行时，我们可用让FPGA并行复制。因为下行和等待都是相同的，不需要重复多次操作。只有上行才有不同，需要分开处理。这里，我们充分发挥FPGA的并行处理能力。把上行数据并行接收，存入先进先出（FIFO）的队列中，让MCU来统一读取和处理。

这样一来，多路编程几乎是并行的，效率大大提高。单次单路FPGA编程大约需要25秒左右，当我们扩展出8路，对8路进行并行编程，总共才需要不到28秒的时间，效率极高。

编程文件保护

FPGA的设计是客户的知识产权，所以需要对FPGA的编程bin文件进行保护。客户把编程bin文件拷贝到TF卡上，插到编程器上后，编程器使用HASH，AES等算法对文件进行加密之后再保存到FLASH里面。以保护编程文件不被拷贝和滥用。

编程器还提供编程次数限制功能。客户可以设置编程次数。编程次数达到限制后，编程器会锁定，不再可用。客户通过TF卡升级编程文件，设置编程次数后，可以放心的把编程器交到生产线，不用担心编程文件被拷贝和滥用。

总结

  • 利用FPGA的并行能力，成倍提高编程效率。
  • 利用加密算法保护编程文件，提高安全和可用性。
  • 利用了STM32的大多数基本功能。
